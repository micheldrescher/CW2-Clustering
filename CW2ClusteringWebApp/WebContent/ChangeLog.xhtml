1.4 - 8 April 2016

  • Functionality - Include unvalidated responses
    Added an option allowing users to also include unvalidated submissions in their analyses. This allows users to perform analyses straight away seeing where their activity sits, at the risk of the analysis being tainted by illegitimate, silly submissions that haven't been weeded out yet. Since the use of the application is entirely anonymous, we cannot distinguish which submissions originated from which of the current users to filter out all other, unrelated unvalidated submissions.
  • Functionality - Allow custom significance thresholds
    By default, the application uses the Kaiser-Guttman criterion (i.e. a value of 1.0) to determine which and how many of the selected characteristics should be considered significant for further analysis. Some criticism argues that the Kaiser-Guttman criterion is not always applicable. We thus allow users to decide for themselves which threshold should be applied. The default threshold remains Kaiser-Guttman.
  • Functionality - Allow option to skip projecting project scores onto significant characteristics
    Projecting scores onto the significant dimensions (i.e. characteristics) emphasises the variance of responses related to these dimensions, but dampens the variance among the non-significant dimensions. If a user's activity happens to be scored high in non-significant characteristics, it might be misrepresented in the follow-on clustering phase. Not projecting scores onto significant characteristics eliminates this underrepresentation at the risk of a much more scattered clustering result.
